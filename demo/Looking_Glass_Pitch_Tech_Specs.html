Technical · Pipeline
DEEP-DIVE

The pipeline

  • YOLO-World — open-vocabulary detection, any object you can describe
  • ByteTrack — multi-object tracking with persistent IDs across frames
  • SigLIP — encodes frames and queries into a shared 768-dim vector space
  • Florence-2 — dense scene captions with spatial grounding per frame
  • Qdrant — embedded vector store, sub-second cosine similarity search
  • Llama 3.2 3B via Ollama — local LLM for analytics chat, fully on-device

~2,000 lines Python · ~550 lines React · FastAPI + Vite · Built in 2 weeks

Technical · Architecture
DEEP-DIVE

End-to-end flows

  • Ingestion: MP4 clips → decord sampling (1 FPS) → detection + tracking → embedding + captioning → vector store
  • Search: Query → SigLIP text embedding → Qdrant cosine search → re-ranking with caption cross-check → top-K with bounding boxes
  • Alerts: NL query registered → continuous frame scanning → WebSocket push on match → real-time notification
  • Analytics: Question → Llama 3.2 interprets → track-based counting → conversational answer
  • Frontend: React + Vite dark UI · multi-camera grid · canvas bbox overlays · chat panel

Technical · Model Choices
DEEP-DIVE

Why these models

YOLO-World over YOLOv8

Open vocabulary: detects any described object without retraining. No fixed class list.

SigLIP over CLIP

Sigmoid loss gives better zero-shot accuracy. More efficient than contrastive softmax at scale.

Florence-2 over BLIP-2

Denser captions with spatial grounding. Better retrieval via richer text anchors.

Qdrant over Pinecone

Embedded mode — no server process. Runs in-process, fully offline, zero infrastructure.

Ollama over OpenAI

100% local inference. No API keys. GDPR-compliant. Works in airplane mode.

React+Vite over Next.js

Static SPA is sufficient. No SSR overhead. Faster dev cycle for a demo-scope project.

Technical · Limitations
DEEP-DIVE

Current limitations

Pre-recorded only

Architecture supports live webcam/RTSP (stubs exist), but demo uses MP4 clips. Real-time ingestion needs GPU headroom.

Single device

Currently runs on one machine. Distributed ingestion and federated search are planned but not implemented.

Open-vocab accuracy

Semantic matching is not 100%. Rare or highly abstract queries can miss. Rescue tactics: prompt tuning, caption boosting, manual hints.

Latency on CPU

GPU (Apple Silicon MPS or NVIDIA CUDA) is strongly recommended. CPU fallback works but search latency rises to 2–5 seconds.

No facial recognition

By design — GDPR. But this means person re-identification relies on clothing/appearance, which breaks across days.

LLM hallucination

Llama 3.2 3B is small. Analytics answers are grounded in track data but can misinterpret edge-case questions.
